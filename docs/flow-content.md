## 流程连线与分支

- 需求驱动（补充）：需求提出 -> 需求评审
- 需求评审 -> 需求拆解：评审通过
- 需求评审 -> 需求提出：评审不通过，补充需求材料
- 需求拆解 -> 数据探索：不可合并，新模型开发
- 需求拆解 -> 模型设计：可合并，迭代已有模型
- 机理驱动（主导）：从数据探索进入主流程
- 数据探索 -> 特征确认
- 特征确认 -> 模型设计
- 模型设计 -> 模型评审
- 模型评审 -> 测试验证：评审通过
- 模型评审 -> 模型设计：评审不通过
- 测试验证 -> 部署上线：测试通过
- 测试验证 -> 模型设计：测试不通过
- 部署上线 -> 运营监测
- 运营监测 -> 模型迭代：需要迭代
- 运营监测 -> 持续运营监测：无需迭代
- 模型迭代 -> 模型设计：进入下一轮优化

## 点击节点展开内容

### 1. 需求提出

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 将需求文档转成结构化任务，抽取部件对象、风险模式、指标口径和验收目标。

**人工判断**

- 补充需求背景，确认需求方和算法侧接口。

**交付物**

- 需求评审输入卡

### 2. 需求评审

#### 需求来源 B：需求驱动（补充）

**人工判断**

- 判断需求价值、数据可行性、业务边界和优先级。

**判断分支**

- 评审通过：进入需求拆解。
- 评审不通过：退回需求提出补充材料。

**交付物**

- 需求拆解输入卡

### 3. 需求拆解

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 拆解风险场景、字段指标和模型复用可能性。
- 检索已有模型，判断是新增开发还是已有模型迭代。

**人工判断**

- 确认拆解口径和复用判断是否符合业务诉求。

**判断分支**

- 可合并：进入模型设计，迭代已有模型。
- 不可合并：进入数据探索，开发新模型。

**交付物**

- 数据探索或模型设计输入卡

### 4. 数据探索

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 围绕机理假设自动拉取相关样本，完成**趋势、分布、异常片段**探索。
- 识别可用于建模的信号组合和数据质量问题。

**交付物**

- 候选特征输入包

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 自动完成字段覆盖、缺失、异常样本和工况分布探索。

**交付物**

- 特征确认输入包

### 5. 特征确认

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 根据数据探索结果生成候选特征，并筛选与风险机理一致的关键特征。

**交付物**

- 模型设计输入特征表

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 将需求指标和数据探索结果转成可计算特征，并确认字段口径。

**交付物**

- 模型设计输入特征表

### 6. 模型设计

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 基于已确认特征生成模型或规则方案，输出阈值、窗口、屏蔽条件和风险等级草案。

**交付物**

- 模型评审输入包

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 生成模型或规则方案，包含触发条件、屏蔽条件、风险等级和参数建议。

**交付物**

- 模型评审输入包

### 7. 模型评审

#### 需求来源 A：机理驱动（主导）

**人工判断**

- 确认模型解释性、风险等级、误报漏报权衡和上线边界。

**判断分支**

- 评审通过：进入测试验证。
- 评审不通过：回到模型设计。

**交付物**

- 测试验证输入方案

#### 需求来源 B：需求驱动（补充）

**人工判断**

- 确认业务解释性、上线风险、处置动作和责任边界。

**判断分支**

- 评审通过：进入测试验证。
- 评审不通过：回到模型设计。

**交付物**

- 测试验证输入方案

### 8. 测试验证

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 自动生成测试配置、回放脚本和边界样本验证结果。

**判断分支**

- 测试通过：进入部署上线。
- 测试不通过：回到模型设计修正。

**交付物**

- 上线输入包

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 自动生成测试用例、回放结果和边界工况检查。

**判断分支**

- 测试通过：进入部署上线。
- 测试不通过：回到模型设计修正。

**交付物**

- 上线输入包

### 9. 部署上线

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 生成发布配置、上线清单和版本记录。

**交付物**

- 运营监测配置

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 生成发布配置、上线清单和版本记录。

**交付物**

- 运营监测配置

### 10. 运营监测

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 自动监测触发效果、人工确认率、误报样本和漏报线索。

**判断分支**

- 需要迭代：进入模型迭代。
- 无需迭代：持续运营监测。

**交付物**

- 模型迭代输入

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 自动汇总触发效果、需求方反馈、误报样本和现场处置结果。

**判断分支**

- 需要迭代：进入模型迭代。
- 无需迭代：持续运营监测。

**交付物**

- 模型迭代输入

### 11. 模型迭代

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 汇总线上反馈，生成模型优化方案和版本变更建议。

**人工判断**

- 确认是否进入新一轮模型设计或补充数据探索。

**交付物**

- 下一轮模型设计输入

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 根据运营反馈生成字段补充、阈值调整、屏蔽条件和版本优化建议。

**人工判断**

- 确认是否满足需求验收口径，以及是否进入下一轮模型设计。

**交付物**

- 下一轮模型设计输入

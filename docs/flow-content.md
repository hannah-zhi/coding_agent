## 流程连线与分支

页面展示约束：所有分支连线采用肘形箭头，避免曲线回路造成标签和箭头对应关系不清。

- 需求驱动（补充）：需求提出 -> 需求评审
- 需求评审 -> 需求拆解：评审通过
- 需求评审 -> 需求提出：评审不通过
- 需求拆解 -> 数据探索：不可合并，新模型开发
- 需求拆解 -> 模型迭代：可合并，迭代已有模型
- 机理驱动（主导）：从数据探索进入主流程
- 数据探索 -> 模型设计
- 模型设计 -> 模型评审
- 模型评审 -> 测试验证：评审通过
- 模型评审 -> 模型设计：评审不通过
- 测试验证 -> 部署上线：测试通过
- 测试验证 -> 模型设计：测试不通过
- 部署上线 -> 运营监测
- 运营监测 -> 模型迭代：需要迭代
- 运营监测 -> 运营监测：无需迭代，持续监测
- 模型迭代 -> 模型设计：版本更新

## 点击节点展开内容

### 1. 需求提出

#### 需求来源 B：需求驱动（补充）

**人工介入**

- 挖掘业务痛点，梳理相应的模型需求。

**Agent 执行**

- 将自然语言描述的模型需求转化为结构化的需求文档。

**交付物**

- 初版BRD需求文档（Markdown）

### 2. 需求评审

#### 需求来源 B：需求驱动（补充）

**人工介入**

#### 需求来源 B：需求驱动（补充）

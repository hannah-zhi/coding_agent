- 需求来源 A：电池机理驱动

## 角色与颜色

- Agent：青色节点边框，对应 Agent 自动化环节。
- 人工：琥珀色节点边框，对应人工确认、评审、决策环节。

      - 完成迭代：更新已有模型

## 流程连线与分支

- 需求进入 -> 场景理解与复用判断
- 场景理解与复用判断 -> 数据提取：不可合并，新模型开发
- 场景理解与复用判断 -> 模型/规则设计：可合并，迭代已有模型
- 数据提取 -> EDA 探索
- EDA 探索 -> 特征发现：电池路径
- EDA 探索 -> 字段/指标映射：CBB 路径
- 特征发现 / 字段指标映射 -> 需求/机理确认
- 需求/机理确认 -> 模型/规则设计
- 模型/规则设计 -> 算法评审
- 算法评审 -> 代码测试
- 代码测试 -> 上线监控
- 上线监控 -> 模型迭代闭环
- 模型迭代闭环 -> 需求进入：发现新问题，进入新一轮流程

## 点击节点展开内容

### 1. 需求进入

#### 需求来源 A：电池机理驱动

**人工判断内容**

- 判断该风险是否具备安全价值和开发优先级。
- 确认是否进入电池安全预警算法开发流程。

**明确交付物**

- 电池风险任务卡
- 失效模式描述
- 样本筛选口径
- 优先级建议

#### 需求来源 B：CBB 部件需求驱动

**人工判断内容**

- 判断 CBB 需求文档是否具备算法化条件。
- 确认需求方、部件专家和算法侧的接口人。
- 明确期望识别的风险类型、处置动作和上线边界。

**明确交付物**

- CBB 需求任务卡
- 需求文档结构化摘要
- 待澄清问题清单
- 需求方确认记录

### 2. 场景理解与复用判断

#### 需求来源 A：电池机理驱动

**Agent 执行内容**

- 围绕内短、异常衰退、一致性劣化等风险模式整理机理链和特征假设。
- 检索已有电池安全模型，对比风险对象、输入特征、触发逻辑和适用边界。
- 输出“可合并 / 不可合并”的证据清单。

**人工判断内容**

- 判断该需求是否能合并入已有模型。
- 如果机理一致且只需新增特征或调参，则进入已有模型迭代。
- 如果风险模式不同，则进入新模型开发。

**判断分支**

- 可合并：复用已有模型，后续做特征补充、阈值调整或版本迭代。
- 不可合并：作为新模型 / 新规则进入完整开发流程。

**明确交付物**

- 模型复用判断卡
- 已有模型差异对比表
- 新建 / 迭代路径建议

#### 需求来源 B：CBB 部件需求驱动

**Agent 执行内容**

- 将 CBB 需求文档转成风险场景、指标定义、异常口径和所需字段。
- 检索已有高压箱、绝缘、接触器、环控类模型或规则。
- 对比已有能力是否覆盖新需求。

**人工判断内容**

- 需求方确认该需求是新增风险、已有风险扩展，还是已有模型维护。
- 部件专家确认指标含义和异常口径。

**判断分支**

- 可合并：纳入已有 CBB 部件模型或规则。
- 不可合并：进入新模型或新规则开发。

**明确交付物**

- CBB 场景解析卡
- 复用判断表
- 需求澄清问题清单

### 3. 数据提取

#### 需求来源 A：电池机理驱动

**Agent 执行内容**

- 自动生成面向电芯、簇、站点的 SQL 和样本筛选条件。
- 批量拉取单体电压、温度、SOC、压差、告警等时序数据。

**明确交付物**

- 电池时序数据集
- SQL 脚本
- 样本标签表
- 数据质量初检表

#### 需求来源 B：CBB 部件需求驱动

**Agent 执行内容**

- 根据需求文档生成字段查询方案。
- 拉取接触器状态、绝缘电阻、高压箱温度、告警、工单等数据。
- 输出字段缺失、采样频率、数据连续性检查结果。

**明确交付物**

- CBB 数据集
- 字段映射表
- SQL 脚本
- 数据可用性检查表

### 4. EDA 探索

#### 需求来源 A：电池机理驱动

**Agent 执行内容**

- 输出趋势图、分布统计、异常片段、相关性摘要。
- 对正常样本与疑似风险样本做批量对比，定位可用信号。

**明确交付物**

- EDA 自动报告
- 异常样本清单
- 关键图表
- 数据可用性结论

#### 需求来源 B：CBB 部件需求驱动

**Agent 执行内容**

- 输出部件指标趋势、状态切换、温升分布、绝缘波动、告警前后对比。
- 分析字段是否能支撑需求文档中的异常定义。

**明确交付物**

- CBB EDA 报告
- 字段可用性结论
- 异常片段截图
- 需求方确认材料

### 5. 特征发现

#### 需求来源 A：电池机理驱动

**Agent 执行内容**

- 生成电芯一致性、压差扩散、温升斜率、异常恢复等候选特征。
- 对候选特征做正负样本对比和稳定性检查。
- 形成可进入模型/规则设计的特征候选清单。

**明确交付物**

- 特征候选表
- 特征计算口径
- 特征有效性证据
- 保留 / 修改 / 废弃建议

### 6. 字段/指标映射

#### 需求来源 B：CBB 部件需求驱动

**Agent 执行内容**

- 将 CBB 需求文档中的指标映射到生产数据库字段。
- 标注字段来源、采样频率、缺失情况、单位口径和可替代字段。

**明确交付物**

- 指标字段映射表
- 字段质量检查表
- 可替代字段建议
- 待需求方确认项

### 7. 需求/机理确认

#### 需求来源 A：电池机理驱动

**人工判断内容**

- 电池专家确认机理链、风险等级、处置口径是否成立。
- 判断异常片段是否具有真实工况意义。
- 冻结后续模型开发的关键定义。

**明确交付物**

- 机理确认记录
- 冻结后的风险定义
- 评审输入材料

#### 需求来源 B：CBB 部件需求驱动

**人工判断内容**

- CBB 需求方确认字段、指标、异常定义和业务解释。
- 部件专家确认接触器、绝缘、高压箱温升等风险口径。
- 冻结算法开发边界和验收口径。

**明确交付物**

- 需求确认记录
- 字段口径确认表
- 异常定义冻结版本
- 评审输入材料

### 8. 模型/规则设计

#### 需求来源 A：电池机理驱动

**Agent 执行内容**

- 根据 EDA 与机理确认结果生成规则 / 模型原型。
- 自动给出阈值建议、组合条件、风险等级解释、方案对比。

**明确交付物**

- 算法逻辑草案
- 风险等级解释
- 阈值 / 窗口参数
- 方案对比表

#### 需求来源 B：CBB 部件需求驱动

**Agent 执行内容**

- 将 CBB 异常定义转化成规则、阈值、状态机或评分逻辑。
- 自动生成屏蔽条件、工况过滤、告警等级和解释口径。

**明确交付物**

- CBB 算法逻辑草案
- 阈值和参数建议
- 屏蔽条件清单
- 方案对比表

### 9. 算法评审

#### 需求来源 A：电池机理驱动

**人工判断内容**

- 评审机理合理性、误报漏报、上线边界和处置价值。
- 决定灰度上线、修改后再评审或暂停。

**明确交付物**

- 评审 PPT / 报告
- 评审结论
- 修改意见清单
- 上线约束

#### 需求来源 B：CBB 部件需求驱动

**人工判断内容**

- 需求方和部件专家评审业务解释性和现场处置可行性。
- 确认规则是否会误伤维护、检修、特殊运行工况。

**明确交付物**

- CBB 评审材料
- 评审结论
- 需求方确认意见
- 上线风险约束

### 10. 代码测试

#### 需求来源 A：电池机理驱动

**Agent 执行内容**

- 生成代码草稿、参数配置、单测用例和样本回放脚本。
- 自动检查边界输入、缺失字段、异常工况和配置一致性。

**明确交付物**

- 代码草稿
- 参数配置文件
- 单测用例
- 回放测试结果

#### 需求来源 B：CBB 部件需求驱动

**Agent 执行内容**

- 生成 CBB 规则代码、字段映射配置、单测和回放脚本。
- 检查字段缺失、状态枚举、单位转换和异常分支。

**明确交付物**

- CBB 代码草稿
- 字段映射配置
- 单测用例
- 回放测试结果

### 11. 上线监控

#### 需求来源 A：电池机理驱动

**Agent 执行内容**

- 自动汇总触发次数、确认率、误报样本、漏报线索。
- 生成阈值调整和下一版优化建议。

**明确交付物**

- 运行监控日报 / 周报
- 误报漏报样本清单
- 迭代 Backlog
- 版本优化建议

#### 需求来源 B：CBB 部件需求驱动

**Agent 执行内容**

- 汇总 CBB 规则触发、需求方确认、工单闭环和现场反馈。
- 识别维护工况、检修动作、采集异常带来的误报。

**明确交付物**

- CBB 运行监控报告
- 需求方反馈清单
- 误报样本清单
- 迭代建议

### 12. 模型迭代闭环

#### 需求来源 A：电池机理驱动

**Agent 执行内容**

- 汇总本轮线上效果、评审意见和样本复盘，生成模型迭代方案。
- 更新已有模型的特征版本、阈值参数、屏蔽条件和测试集。

**人工判断内容**

- 判断迭代是否完成、是否需要重新评审、以及是否产生新的风险假设。

**判断分支**

- 完成迭代：更新模型库和版本记录。
- 发现新风险：沉淀为新需求，回到需求进入开启下一轮。

**明确交付物**

- 模型版本更新记录
- 迭代变更说明
- 新一轮需求 Backlog

#### 需求来源 B：CBB 部件需求驱动

**Agent 执行内容**

- 汇总 CBB 模型或规则的触发效果、需求方反馈和现场样本。
- 生成字段补充、阈值调整、屏蔽条件优化和规则合并建议。

**人工判断内容**

- 需求方判断迭代是否满足业务目标。
- 部件专家确认是否需要调整风险定义或处置动作。

**判断分支**

- 完成迭代：更新 CBB 规则库和版本记录。
- 发现新问题：回到需求进入，形成新需求或新场景。

**明确交付物**

- CBB 规则版本记录
- 迭代复盘报告
- 新需求 Backlog

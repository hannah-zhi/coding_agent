## 流程连线与分支

页面展示约束：所有分支连线采用肘形箭头，避免曲线回路造成标签和箭头对应关系不清。

- 需求驱动（补充）：需求提出 -> 需求评审
- 需求评审 -> 需求拆解：评审通过
- 需求评审 -> 需求提出：评审不通过，补充需求
- 需求拆解 -> 数据探索：不可合并，新模型开发
- 需求拆解 -> 模型迭代：可合并，迭代已有模型
- 机理驱动（主导）：从数据探索进入主流程
- 数据探索 -> 模型设计
- 模型设计 -> 模型评审
- 模型评审 -> 测试验证：评审通过
- 模型评审 -> 模型设计：评审不通过，回到设计
- 测试验证 -> 部署上线：测试通过
- 测试验证 -> 模型设计：测试不通过，回到设计
- 部署上线 -> 运营监测
- 运营监测 -> 模型迭代：需要迭代
- 运营监测 -> 运营监测：无需迭代，持续监测
- 模型迭代 -> 模型设计：下一轮优化

## 点击节点展开内容

### 1. 需求提出

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 将自然语言描述的模型需求转化为结构化的需求文档（Markdown）。

**人工判断**

- 挖掘业务痛点，梳理相应的模型需求。

**交付物**

- 初版BRD需求文档（Markdown）

### 2. 需求评审

#### 需求来源 B：需求驱动（补充）

**人工判断**

- 判断需求价值、数据可行性、业务边界和优先级。

**判断分支**

- 评审通过。
- 评审不通过：重新梳理需求。

**交付物**

- 冻结版BRD需求文档（Markdown）

### 3. 需求拆解

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 拆解风险场景、测点参数，检索已有模型，判断模型复用可能性。

**人工判断**

- 确认需求拆解和复用判断是否符合业务诉求。

**判断分支**

- 可合并：进入模型迭代，迭代已有模型。
- 不可合并：进入数据探索，开发新模型。

**交付物**

- PRD需求文档（Markdown）
- 模型开发排期

### 4. 数据探索

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 依据机理分析假设，利用伽利略codex自动提取云端数据中相关测点，进行假设中特征、趋势、分布等探索性分析及验证。

**交付物**

- 数据探索报告（html）
- 数据探索计算脚本（py）

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 基于PRD需求文档，利用伽利略codex自动提取云端数据中相关测点，进行需求中特征、趋势、分布等探索性分析及验证。

**交付物**

- 数据探索报告（html）
- 数据探索计算脚本（py）

**Agent 执行**

- 基于数据探索结果构建模型设计方案，包含主逻辑、阈值、窗口、屏蔽条件和风险等级等技术细节。

**交付物**

- 初版模型设计文档（Markdown）

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 基于数据探索结果构建模型设计方案，包含主逻辑、阈值、窗口、屏蔽条件和风险等级等技术细节。

**交付物**

- 初版模型设计文档（Markdown）

### 6. 模型评审

#### 需求来源 A：机理驱动（主导）

**人工判断**

- 人工评审模型设计方案的机理合理性、可解释性、误报漏报权衡和上线边界。

**判断分支**

- 评审通过：进入测试验证。
- 评审不通过：回到模型设计。

**交付物**

- 冻结版模型设计文档（Markdown）

#### 需求来源 B：需求驱动（补充）

**人工判断**

- 人工评审模型设计方案的业务解释性、上线风险、处置动作和责任边界。

**判断分支**

- 评审通过：进入测试验证。
- 评审不通过：回到模型设计。

**交付物**

- 冻结版模型设计文档（Markdown）

### 7. 测试验证

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

**判断分支**

- 测试通过：进入部署上线。
- 测试不通过：回到模型设计修正。

**交付物**

- 模型测试报告（html）
- 测试回放脚本（py）

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

**判断分支**

- 测试通过：进入部署上线。
- 测试不通过：回到模型设计修正。

**交付物**

- 模型测试报告（html）
- 测试回放脚本（py）

### 8. 部署上线

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 基于测试通过结果生成发布配置、上线清单和版本记录。

**交付物**

- 上线版本记录

#### 需求来源 B：需求驱动（补充）

### 9. 运营监测

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 自动监测模型触发效果、人工确认率、误报样本和漏报线索，形成运行复盘。

**判断分支**

- 需要迭代：进入模型迭代。
- 无需迭代：持续运营监测。

**交付物**

- 模型运行监测报告（html）

**判断分支**

- 无需迭代：持续运营监测。

**交付物**

- 模型运行监测报告（html）

### 10. 模型迭代

#### 需求来源 A：机理驱动（主导）

**Agent 执行**

- 汇总线上反馈和运行监测结果，生成模型优化方案和版本变更建议。

**人工判断**

- 人工确认迭代范围、优先级和是否进入下一轮模型设计。

**交付物**

- 模型迭代方案（Markdown）

#### 需求来源 B：需求驱动（补充）

**Agent 执行**

- 根据运营反馈生成字段补充、阈值调整、屏蔽条件和版本优化建议。

**人工判断**

- 人工确认迭代范围、验收口径和是否进入下一轮模型设计。

**交付物**

- 模型迭代方案（Markdown）
